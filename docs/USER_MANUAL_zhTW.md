# TextileSearch — 使用手冊

**版本 1.0 · 繁體中文**

English version: [USER_MANUAL.md](./USER_MANUAL.md)

---

## TextileSearch 是什麼？

TextileSearch 是一款桌面應用程式，可讓您透過視覺相似度搜尋布料圖片資料庫。將任何布料照片拖放到應用程式中，即可立即在您的收藏中找到視覺上最相似的布料，完全不需要網路連線。

此外，應用程式還能讀取布料樣品上的標籤文字（供應商名稱、成分、幅寬、重量），並可依這些數值篩選結果。

所有資料均儲存於您的電腦中，不會傳送至任何地方。

---

## 安裝

1. 從 Releases 頁面下載 `TextileSearch-Setup-*.exe`
2. 雙擊安裝程式。若 Windows 顯示安全性警告，請點選**更多資訊** → **仍要執行**。此警告是因為應用程式尚未向 Microsoft 登記；這是安全的。
3. 依照安裝步驟操作。大多數使用者使用預設安裝路徑即可。
4. 從開始功能表或桌面捷徑啟動 TextileSearch。

**首次啟動比之後慢。** 應用程式正在建立資料庫並首次載入 AI 模型，約需 30–60 秒。之後啟動只需 5 秒以內。

**安裝檔大小：約 1.3 GB。** 這是正常現象——所有用於布料比對與標籤辨識的 AI 模型均已包含在內，無需額外下載。

## 介面說明

應用程式左側側邊欄提供六個功能區：

| 功能區 | 說明 |
|---|---|
| **搜尋** | 依視覺相似度尋找布料 |
| **圖庫** | 以篩選條件瀏覽完整資料庫 |
| **匯入** | 新增布料圖片資料夾 |
| **歷史記錄** | 重新開啟過去的搜尋 |
| **重複項目** | 審查被標記的重複圖片 |
| **設定** | 語言、搜尋預設值、顯示選項 |

## 步驟一：匯入圖片

搜尋前，應用程式需要先建立您的布料圖片索引。

1. 點選側邊欄的**匯入**
2. 點選**新增資料夾**
3. 選擇包含布料圖片的根資料夾

應用程式會自動掃描該資料夾及所有子資料夾，您無需事先整理圖片——任何資料夾結構皆可。

**匯入過程：**
- 計算每張圖片的視覺特徵向量（embedding）
- 使用 OCR 從每張圖片擷取標籤文字
- 產生縮圖供顯示用
- 所有資料儲存於本地資料庫

**匯入需要多久？**
- 僅使用 CPU：約每秒 5 張圖片
- 使用 GPU：約每秒 30 張圖片
- 10,000 張圖片的資料庫在 CPU 上約需 30–35 分鐘

匯入期間仍可使用應用程式——已建立索引的圖片可立即搜尋和瀏覽。

**新增更多資料夾：** 可以新增任意數量的資料夾，再次點選**新增資料夾**即可。

**應用程式絕不刪除圖片。** 它只讀取圖片，不會修改或移動。若要從資料庫中移除圖片，在檔案總管中刪除或移動即可——應用程式會自動偵測變更。

---

## 步驟二：以圖片搜尋

1. 點選側邊欄的**搜尋**
2. 選擇以下任一方式：
   - 將布料圖片從檔案總管**拖放**至放置區域，或
   - **點選**放置區域開啟檔案選擇器
3. 應用程式會找出視覺上最相似的布料並以網格顯示

**調整結果數量：** 使用**最多結果數**滑桿顯示 5–100 個結果。數量越多，找到的符合項目越多，但相似度可能較低。

**相似度標籤：**
- 🟢 綠色（≥ 90%）——非常相似
- 🟡 琥珀色（70–89%）——中度相似
- 🔴 紅色（< 70%）——有些相似

**點選結果卡片**會在右側開啟詳細資訊面板，顯示：
- 從標籤擷取的完整元資料（供應商、成分、幅寬、重量）
- 信心指標（綠色 = 高信心，琥珀色 = 建議驗證）
- 從資料夾結構衍生的標籤
- **搜尋相似** — 以此結果為新的查詢圖片
- **在資料夾中顯示** — 開啟檔案總管並反白顯示此檔案

---

## 步驟三：瀏覽與篩選

點選**圖庫**可在不使用查詢圖片的情況下瀏覽完整資料庫。

**可用的篩選條件：**
- 輸入文字以依品項編號或資料夾標籤篩選
- 依日期、檔名或重量排序
- 切換顯示或隱藏遺失的檔案（已移動或刪除的圖片）

**多選：** 點選圖片即可選取，已選取的圖片會顯示勾選標記。

**分頁：** 圖庫每頁顯示 100 張圖片，使用底部的 ← → 箭頭翻頁。

## 了解元資料標籤

應用程式讀取標籤文字時，會為每個擷取的欄位分配信心等級。

| 標籤 | 含義 | 建議處理方式 |
|---|---|---|
| 無標籤 | 高信心（≥ 90%）| 可信任 |
| **審核** 琥珀色標籤 | 中等信心（65–89%）| 手動驗證數值 |
| 未顯示 | 低信心（< 65%）| 未擷取此欄位 |

**成分加總檢查：** 若成分百分比加總不等於 100%（±2% 容差），無論 OCR 信心多高，該欄位一律標記為需審核。這是刻意設計的——標示 87%+10%+2%+5% = 104% 的標籤很可能有誤讀。

## 資料夾標籤

應用程式會自動從您的資料夾結構建立標籤。例如：

```
/布料/
    羊毛/
        2025冬季/
            swatch_001.jpg    → 標籤：羊毛、2025冬季
    棉/
        夏季/
            swatch_002.jpg    → 標籤：棉、夏季
```

這些標籤以灰色藥丸形狀顯示於詳細資訊面板，可用於篩選圖庫。標籤為唯讀——在檔案總管中移動檔案時標籤會自動更新。

## 重複項目

點選**重複項目**審查應用程式標記為重複的圖片。

偵測到兩種類型的重複項目：

- **完全相同** — 相同檔案（像素內容相同，但檔名或位置不同）
- **視覺相似** — 視覺上非常相似（預設相似度 ≥ 97%）

對於每組配對，您可以：
- 並排查看兩張圖片
- 查看檔案大小和日期以決定保留哪個
- 點選**在資料夾中顯示**開啟檔案總管
- 點選**標記為已解決**從清單中關閉此配對

**應用程式不刪除檔案。** 若要實際移除重複項目，在檔案總管中刪除即可。應用程式將在下次啟動時偵測到刪除並移除孤立記錄。

---

## 歷史記錄

點選**歷史記錄**查看所有過去的搜尋，包含查詢圖片和最佳結果。點選**重新執行**可重複任何搜尋。

歷史記錄預設保留 12 個月，可隨時從歷史記錄頁面清除。

## 設定

| 設定項目 | 預設值 | 說明 |
|---|---|---|
| 語言 | 繁體中文 | 即時切換，無需重新啟動 |
| 預設結果數量 (k) | 20 | 搜尋預設顯示幾個結果 |
| 重複偵測閾值 | 0.97 | 數值越低，找到的重複配對越多 |
| 顯示遺失圖片 | 關閉 | 顯示已移動/刪除檔案的圖片 |
| 縮圖快取大小 | 2 GB | 縮圖使用的磁碟空間 |

## 自動檔案監控

新增資料夾後，應用程式會持續監控。複製、移動或儲存到資料夾的任何圖片都會在幾秒內自動建立索引，無需手動重新匯入。

若重新命名或移動資料夾，應用程式會偵測到變更並更新記錄——不會遺失任何資料。

## 如果出現問題

**應用程式無法啟動：**
- 確保通過執行 `.exe` 安裝程式來安裝，而非直接複製資料夾
- 嘗試重新啟動電腦

**匯入後圖片未出現：**
- 查看底部狀態列——若顯示「匯入中」表示程序仍在執行
- 若匯入卡住，點選**匯入** → **暫停** → **繼續**

**搜尋結果不正確：**
- 確保查詢圖片是布料照片，而非花紋或文件
- AI 模型最適合在素色背景前拍攝的布料樣品

**未讀取文字標籤：**
- OCR 最適合清晰、平整、正面拍攝的標籤
- 非常小、模糊或高度旋轉的文字可能無法正確辨識
- 可在詳細資訊面板中查看原始 OCR 文字

**記錄檔案：**
- 點選設定 → **開啟日誌資料夾**查看詳細記錄
- 記錄為純文字格式，每 30 天重置一次

## 隱私與資料

- 所有資料僅儲存於您的電腦
- 不傳送任何圖片、元資料或使用資料至任何伺服器
- AI 模型完全在本地執行——安裝後無需網路連線
- 解除安裝應用程式不會刪除您的圖片或 TextileSearch 資料夾。若要完全移除所有資料，解除安裝後請刪除 `%APPDATA%\TextileSearch`
